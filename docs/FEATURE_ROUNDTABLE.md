# Feature Roundtable — Pattern Reset™

> A multi-disciplinary panel to evaluate and shape new features, pages, and functionality for Pattern Reset. Use this when adding a new feature, page, or experience to the product.

---

## Roundtable Members

| Role | Focus | Key Questions They Ask |
|------|--------|------------------------|
| **Product Lead** | Vision, roadmap fit, user value, prioritization | Does this serve our core audience? Does it fit the Entry → Root → Outcome journey? What's the success metric? |
| **Engineer** | Feasibility, architecture, performance, security | Can we build it with our stack? What's the data model? How does it integrate with existing systems (Firebase, Stripe, localStorage)? |
| **Designer** | Visual consistency, brand alignment, layout | Does it match our design system (cream, red, Anton, DM Sans)? How does it feel across breakpoints? |
| **Marketing & Sales** | Conversion, messaging, funnel placement | Where does this live in the funnel? How do we message it? Does it drive email capture or purchase? |
| **MVP Product Expert** | Scope, speed to launch, must-have vs nice-to-have | What's the smallest version that delivers value? What can we cut for v1? |
| **UI/UX Expert** | Flow, accessibility, usability, delight | Is the flow intuitive? Where do users get stuck? Is it accessible? |
| **Events, Ticketing & Experiences Expert** | Live/immersive experiences, scheduling, community | Could this involve live events, cohorts, or time-bound experiences? How do we handle capacity, scheduling, or ticketing? |

---

## How to Use This Roundtable

1. **Propose your feature** — Describe the new feature, page, or function you want to add.
2. **Roundtable evaluates** — Each expert weighs in with questions, constraints, and recommendations.
3. **Synthesis** — We produce a consolidated view: scope, risks, opportunities, and a recommended path forward.

---

## Quick Reference: Pattern Reset Context

- **Stack:** HTML/CSS/vanilla JS, Node/Express, Firebase, Stripe
- **Audience:** Women 25–44, romantic pattern pain
- **Design:** `#fffcf1` cream, `#ca0013` red, Anton + DM Sans, dark hero gradients
- **Existing pages:** index, patterns, pattern-theory, legal, letitout, product/breakup-course, course/breakup-course
- **Core flow:** Quiz → Email → Results → Workbook → (optional paid products)

## Ready to Start

When you're ready, describe your new feature, page, or function. The roundtable will respond with:

- **Product:** Strategic fit and success criteria  
- **Engineering:** Technical approach and integration points  
- **Design:** Visual and layout guidance  
- **Marketing/Sales:** Messaging and funnel placement  
- **MVP:** Minimal scope for v1  
- **UI/UX:** Flow, usability, and accessibility  
- **Events/Experiences:** Live or time-bound opportunity (if relevant)
